# 评分细则对照 · 本草拾珍

> 对应《信息可视化课程期末作品评分表》。本页说明：**评分维度 → 作品证据 → 主要文件**。

---

## 1. 数据分析与信息叙事能力（20 分）

| 评价参考 | 本作品对应 |
|----------|------------|
| 数据来源与处理 | 本草典 CC BY-SA 开源数据 + 本地扩充 / 方剂 JSON；不爬取受版权保护的《中华本草》全文 |
| 清洗、整理、转换 | `import_herbs.py`、`build_extra_herbs.py`、`expand_to_1000.py`、`fix_bulk_pinyin.py`、`pinyin_fix.py` |
| 统计分析 / 关联 / 趋势 | `analysis.py`：交叉矩阵、桑基、用量分位、风险占比、词频、方剂共现网络；首页朝代累计曲线 |
| 信息叙事逻辑 | **史（朝代轴）→ 方（功效环 / 方义）→ 药（属性图谱 / 详情）**；析图给出文字结论（insight） |
| 数据价值挖掘 | 君臣佐使角色子网、禁忌试炼、用量与风险并读、从症到药词频 |

**关键文件**

- `backend/data/herbs_raw.json` / `herbs_extra.json` / `herbs_bulk.json` / `herbs.db`
- `backend/data/formulas_raw.json` / `formulas_extra.json`
- `backend/data/era_library.json` / `era_maps/` / `herb_geo_density.json`
- `backend/scripts/*.py`（采集、扩充、导入、时代库构建）
- `backend/app/analysis.py`、`era_library.py`

## 2. 可视化设计能力（20 分）

| 评价参考 | 本作品对应 |
|----------|------------|
| 图表选择 | 环图、力导向、热力图、桑基、区间+散点、柱状、饼图等按问题选型 |
| 视觉编码 | 四气冷暖、君臣佐使配色、功效莫兰迪色；角色着色可切换 |
| 信息层级与布局 | 首页「轴 + 环」主构图；属性页「图 + 列表」；析图「主题 Tab + 单屏切图」 |
| 色彩字体排版 | 宣纸风、衬线中文（ZCOOL XiaoWei / Noto Serif SC / Ma Shan Zheng） |

**关键文件**

- `index.html`（样式 + ECharts 配置）
- `china-geo-embed.js`（产区底图嵌入，避免跨域）

## 3. 交互设计能力（15 分）

| 评价参考 | 本作品对应 |
|----------|------------|
| 筛选 / 点击 / 动态 | 属性筛选、朝代切换、环图展开方名、药材详情、产区下钻 |
| 交互逻辑 | 抓药柜 ↔ 炼药炉；试炼选题不泄组成、齐药后融合显方；析图标签切图带过渡 |
| 探索体验 | 悬停标题读结论；云朵配伍提示；药对一键入炉 |

**关键文件**

- `index.html`（前端交互主逻辑）
- `backend/app/alchemy.py`、`workshop.py`（炼药 / 配伍校核 API）

## 4. 技术实现能力与作品完成度（15 分）

| 评价参考 | 本作品对应 |
|----------|------------|
| 工具与技术 | FastAPI + SQLite + ECharts 5；单页前端由后端同源托管 |
| 运行稳定性 | `启动.bat` / `launch.py` 一键起服务；API `/docs` |
| 自主开发 | 自研叙事布局、析图聚合、炼药试炼与时代库，非后台 CMS 模板站 |

**关键文件**

- `启动.bat`、`launch.py`、`index.html`
- `backend/app/main.py` 及同目录模块
- `backend/requirements.txt`

> 说明：本课评分重点为可视化能力；本仓库**无**登录注册后台，符合「不以 CRUD 网站为主要依据」的说明。
